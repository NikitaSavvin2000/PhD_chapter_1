# Система экспериментов временных рядов

## Введение

Репозиторий предназначен для автоматизированного сравнения моделей прогнозирования временных рядов в единой воспроизводимой среде. Используется для сопоставления SOTA и классических методов (деревья решений, случайный лес, LSTM и др.) на одинаковых данных и протоколах.

---

## Быстрый старт

### 1. Установка зависимостей

```bash
pdm install
```

### 2. Активация окружения

```bash
source .venv/bin/activate
```

### 2.1 (При необходимости) настройка PYTHONPATH

```bash
export PYTHONPATH=$PWD/src:$PYTHONPATH
```

### 3. Запуск экспериментов

```bash
pdm run src/runners/main.py
```

### 4. Где смотреть результаты

Все результаты экспериментов автоматически сохраняются в директорию:

```text
export/<experiment_unique_name>/
```

Имя эксперимента задаётся в:

```text
src/runners/params.yaml
```

Внутри директории находятся:

* `experiment_design.csv` — полный список запущенных экспериментов (формируется через `experiment_setup.py`)
* `src_snapshot.zip` — архив состояния `src/` на момент запуска (создаётся в `init_experiment_config`)
* `logs/` — логирование всех этапов выполнения (создаётся через `get_logger`)
* `results/` — результаты моделей (создаются в процессе выполнения пайплайна)

## 1. Инициализация эксперимента

При запуске `main.py` выполняется автоматическая инициализация среды эксперимента.

### 1. Создание директории эксперимента

Формируется структура:

```text
export/<experiment_name>/
```

Если директория уже существует — используется существующая без пересоздания.

---

### 2. Snapshot исходного кода

Создаётся архив текущего состояния кода:

```text
src_snapshot.zip
```

Он фиксирует содержимое директории `src/` и обеспечивает воспроизводимость экспериментов.

---

### 3. Генерация design экспериментов

Файл:

```text
experiment_design.csv
```

Содержит полное пространство экспериментов, сформированное как декартово произведение:

* модель
* датасет
* признаки временного ряда
* интервалы train/test
* имя директории результата

Каждая строка соответствует одному запуску модели.

---

## Логирование

Система использует централизованное логирование на уровне каждого эксперимента.

### Путь хранения логов

```text
export/<experiment_name>/logs/
```

### Что логируется

* инициализация эксперимента;
* создание структуры директорий;
* генерация snapshot исходного кода;
* формирование `experiment_design.csv`;
* ошибки загрузки датасетов;
* ошибки конфигурации;
* прогресс выполнения каждого этапа.

---

## Датасеты

Конфигурация источников данных задаётся в:

```text
src/configs/data_config.py
```

Поддерживаются:

* удалённые CSV (Google Sheets);
* локальные файлы;
* маппинг колонок временных рядов (`time`, `target`, `features`).

---

## Pipeline выполнения экспериментов

После генерации `experiment_design.csv` система выполняет последовательность шагов:

1. Итерация по всем экспериментам из design
2. Загрузка соответствующего датасета
3. Подготовка train/test выборок
4. Запуск модели с заданной конфигурацией
5. Сохранение результатов в:

```text
export/<experiment_name>/results/<result_dir_name>/
```
